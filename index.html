Q: What is Git?
Git is a distributed version control system used to track changes in source code during software development.
Q: Git Fork vs Git Clone
Git Fork creates a personal copy of a GitHub repository, allowing contributions via pull requests. Git Clone creates a local copy of a repository on your computer for development.
Q: When do you use Git Fork instead of Git Clone?
I use Git Fork when contributing to a public project I don't own. It creates a copy in my GitHub account where I can push changes and create a pull request.
Q: How does Git Fork work in practice?
Fork the project → Clone to local → Create a branch → Make changes → Push to your fork → Open a pull request to the original repository.
Q: Git Fetch vs Git Pull
Git Fetch downloads changes but does not apply them. Git Pull fetches and merges the changes into your current branch.
Q: Which do you prefer: Git Fetch or Git Pull?
I prefer Git Fetch to preview incoming changes before merging, which helps avoid conflicts and unexpected updates.
Q: Git Rebase vs Git Merge
Git Merge keeps full history and adds a merge commit. Git Rebase rewrites history by applying your changes on top of another branch for a cleaner history.
Q: How do you handle merge conflicts?
I manually resolve conflicts in the files, stage them with git add, and then commit with git commit. I also use tools like VSCode merge editor.
Q: Have you used Git tags? Why?
Yes, I use Git tags to mark release versions like v1.0.0. It helps track stable points in the project history.
Q: How to combine multiple commits into one?
Use git rebase -i HEAD~n and change "pick" to "squash" for the commits you want to combine into a single commit.
Q: 10 Git commands you use daily?
git clone, git status, git add, git commit, git push, git pull, git fetch, git branch, git checkout, git log
Q: How to prevent a file from being pushed?
Add it to .gitignore. If already tracked, run git rm --cached filename and commit the change.
Q: What is the purpose of the .git folder?
The .git folder contains all Git metadata—commit history, configuration, branches, tags, and object database. It's what makes a directory a Git repository.
Q: Can you restore a deleted .git folder?
Not easily. Once deleted, Git loses all history. Best solution is to re-clone the repo or restore from backup.
Q: What if a secret was accidentally committed to Git?
Remove the file using git rm --cached, rewrite history using BFG or filter-branch, rotate the secret, and force push the clean history.
Q: Explain a scenario where you used Git Fork instead of Git Clone
I used Git Fork when contributing to an open-source project. Since I didn’t have push access, I forked the repository, made my changes, and created a pull request for review.
Q: Git Fork in action with example
Example: Fork the original repo on GitHub → Clone your fork → Create a new branch → Make changes → Push → Create Pull Request on GitHub.
Q: Create a fork of the project and create a pull request
1. Fork the project on GitHub
2. Clone it locally
3. Create a new branch and make changes
4. Push the branch
5. Go to your GitHub repo and open a Pull Request
Q: Show how Git Fetch and Git Pull work in real-time
Use git fetch to get remote changes without applying them. Then use git diff to inspect changes, and git pull to merge them locally.
Q: Which command do you use more – Git Fetch or Git Pull, and why?
I use git fetch more often to inspect changes before merging. It provides control and reduces the risk of unwanted changes breaking the codebase.
Q: Practice Git Fetch vs Git Pull on a GitHub repo
Clone a repo, make changes on GitHub, then run git fetch to bring changes locally, and git pull to apply them. Use git diff origin/main to compare.
Q: Show practically how Git Rebase is different from Git Merge
Use git merge to join histories with a merge commit. Use git rebase to replay your changes on top of the latest branch, producing linear history.
Q: How to explain Git Merge vs Git Rebase in Interviews
Merge preserves history and shows how features combine. Rebase creates cleaner history by placing your changes on top of another branch. Rebase before merging PRs for cleaner logs.
Q: Practice Git Merge vs Git Rebase
Create a feature branch. Make changes and try both git merge main and git rebase main. Observe the differences with git log --graph.
Q: Explain the Git Branching Strategy used in your company
We used Git Flow: main (production), develop (staging), feature branches, hotfix branches. This structured workflow helped manage releases and features cleanly.
Q: Explain 3 challenges faced with Git during work experience
1. Frequent merge conflicts
2. Mistaken force pushes
3. Accidentally pushing sensitive data
Q: Explain the recent Git challenge and how you addressed it
A teammate force-pushed over my commits. I recovered my work using git reflog and cherry-picked the lost changes onto a new branch.
Q: Explain Git Merge Strategies – Ours and Theirs
Ours keeps your current branch’s changes. Theirs keeps the incoming branch’s changes. Use during merge conflicts with: git checkout --ours filename or --theirs.
Q: Create a Merge Conflict and resolve it locally using Git merge
1. Create two branches and edit the same line in both
2. Merge one into the other
3. Git will show a conflict
4. Manually fix it, then git add and git commit
Q: How do you combine multiple commits into a single commit?
Use git rebase -i HEAD~n and mark later commits as squash. This creates a cleaner commit history.
Q: A teammate accidentally committed a Kubernetes Secret – what to do?
1. Remove the secret using git rm --cached
2. Rotate the secret
3. Clean history using BFG or git filter-branch
4. Force push the cleaned branch
Q: What is Docker?
Docker is a containerization platform used to package applications and their dependencies into containers.
Q: What is Kubernetes?
Kubernetes is an open-source platform for automating deployment, scaling, and operations of application containers.
Q: What is Jenkins?
Jenkins is an automation server used to build, test, and deploy code continuously with CI/CD.
Q: What is Terraform?
Terraform is an open-source IaC (Infrastructure as Code) tool that allows you to define and provision infrastructure using a high-level configuration language.

Explore DevOps tools and interview questions

Q: What is Git?
Git is a distributed version control system used to track changes in source code during software development.
Q: Git Fork vs Git Clone
Git Fork creates a personal copy of a GitHub repository, allowing contributions via pull requests. Git Clone creates a local copy of a repository on your computer for development.
Q: When do you use Git Fork instead of Git Clone?
I use Git Fork when contributing to a public project I don't own. It creates a copy in my GitHub account where I can push changes and create a pull request.
Q: How does Git Fork work in practice?
Fork the project → Clone to local → Create a branch → Make changes → Push to your fork → Open a pull request to the original repository.
Q: Git Fetch vs Git Pull
Git Fetch downloads changes but does not apply them. Git Pull fetches and merges the changes into your current branch.
Q: Which do you prefer: Git Fetch or Git Pull?
I prefer Git Fetch to preview incoming changes before merging, which helps avoid conflicts and unexpected updates.
Q: Git Rebase vs Git Merge
Git Merge keeps full history and adds a merge commit. Git Rebase rewrites history by applying your changes on top of another branch for a cleaner history.
Q: How do you handle merge conflicts?
I manually resolve conflicts in the files, stage them with git add, and then commit with git commit. I also use tools like VSCode merge editor.
Q: Have you used Git tags? Why?
Yes, I use Git tags to mark release versions like v1.0.0. It helps track stable points in the project history.
Q: How to combine multiple commits into one?
Use git rebase -i HEAD~n and change "pick" to "squash" for the commits you want to combine into a single commit.
Q: 10 Git commands you use daily?
git clone, git status, git add, git commit, git push, git pull, git fetch, git branch, git checkout, git log
Q: How to prevent a file from being pushed?
Add it to .gitignore. If already tracked, run git rm --cached filename and commit the change.
Q: What is the purpose of the .git folder?
The .git folder contains all Git metadata—commit history, configuration, branches, tags, and object database. It's what makes a directory a Git repository.
Q: Can you restore a deleted .git folder?
Not easily. Once deleted, Git loses all history. Best solution is to re-clone the repo or restore from backup.
Q: What if a secret was accidentally committed to Git?
Remove the file using git rm --cached, rewrite history using BFG or filter-branch, rotate the secret, and force push the clean history.
Q: Explain a scenario where you used Git Fork instead of Git Clone
I used Git Fork when contributing to an open-source project. Since I didn’t have push access, I forked the repository, made my changes, and created a pull request for review.
Q: Git Fork in action with example
Example: Fork the original repo on GitHub → Clone your fork → Create a new branch → Make changes → Push → Create Pull Request on GitHub.
Q: Create a fork of the project and create a pull request
1. Fork the project on GitHub
2. Clone it locally
3. Create a new branch and make changes
4. Push the branch
5. Go to your GitHub repo and open a Pull Request
Q: Show how Git Fetch and Git Pull work in real-time
Use git fetch to get remote changes without applying them. Then use git diff to inspect changes, and git pull to merge them locally.
Q: Which command do you use more – Git Fetch or Git Pull, and why?
I use git fetch more often to inspect changes before merging. It provides control and reduces the risk of unwanted changes breaking the codebase.
Q: Practice Git Fetch vs Git Pull on a GitHub repo
Clone a repo, make changes on GitHub, then run git fetch to bring changes locally, and git pull to apply them. Use git diff origin/main to compare.
Q: Show practically how Git Rebase is different from Git Merge
Use git merge to join histories with a merge commit. Use git rebase to replay your changes on top of the latest branch, producing linear history.
Q: How to explain Git Merge vs Git Rebase in Interviews
Merge preserves history and shows how features combine. Rebase creates cleaner history by placing your changes on top of another branch. Rebase before merging PRs for cleaner logs.
Q: Practice Git Merge vs Git Rebase
Create a feature branch. Make changes and try both git merge main and git rebase main. Observe the differences with git log --graph.
Q: Explain the Git Branching Strategy used in your company
We used Git Flow: main (production), develop (staging), feature branches, hotfix branches. This structured workflow helped manage releases and features cleanly.
Q: Explain 3 challenges faced with Git during work experience
1. Frequent merge conflicts
2. Mistaken force pushes
3. Accidentally pushing sensitive data
Q: Explain the recent Git challenge and how you addressed it
A teammate force-pushed over my commits. I recovered my work using git reflog and cherry-picked the lost changes onto a new branch.
Q: Explain Git Merge Strategies – Ours and Theirs
Ours keeps your current branch’s changes. Theirs keeps the incoming branch’s changes. Use during merge conflicts with: git checkout --ours filename or --theirs.
Q: Create a Merge Conflict and resolve it locally using Git merge
1. Create two branches and edit the same line in both
2. Merge one into the other
3. Git will show a conflict
4. Manually fix it, then git add and git commit
Q: How do you combine multiple commits into a single commit?
Use git rebase -i HEAD~n and mark later commits as squash. This creates a cleaner commit history.
Q: A teammate accidentally committed a Kubernetes Secret – what to do?
1. Remove the secret using git rm --cached
2. Rotate the secret
3. Clean history using BFG or git filter-branch
4. Force push the cleaned branch
Q: What is Linux?
Linux is an open-source operating system kernel widely used in servers, desktops, and embedded systems.
Q: /Var is almost 90 percent full. What will be your next steps?
Identify large files using du -sh * in /var.
Clear old logs using logrotate or remove unwanted files.
Optionally, increase volume size.
Q: Linux Server is slow due to High CPU utilization. How will you fix it?
Use top or htop to identify CPU-heavy processes.
Restart or kill high-usage processes, check for infinite loops or memory leaks.
Q: Application deployed on Nginx returns Connection Refused. How will you fix it?
Check if the application is running and listening on the port.
Verify firewall rules and Nginx config. Check logs for errors.
Q: SSH to an instance stopped working. How will you troubleshoot the issue?
Check security groups, key permissions, and if SSH service is running.
Use EC2 Serial Console or mount the volume to another instance if needed.
Q: Find and list the log files older than 7 days in /var/log folder.
find /var/log -type f -mtime +7
Q: Find and remove the log files older than 30 days in a folder.
find /your/folder -type f -mtime +30 -exec rm -f {} \;
Q: Cronjob + Shell script to perform advanced log rotation.
Create a script to compress and archive logs. Add a cronjob:
0 0 * * * /path/to/your-log-rotate-script.sh
Q: Bulk creation of Linux users using CSV file.
Use a script:
while IFS=, read -r username password; do useradd $username; echo "$username:$password" | chpasswd; done < users.csv
Q: Service Health monitor script in Bash
Use systemctl is-active service_name
If inactive, send alert or restart the service.
Q: Find and delete files over 100MB
find /path -type f -size +100M -exec rm -i {} \;
Q: Get the list of users who logged in today
Use last -F | grep "$(date '+%b %d')" | cut -d ' ' -f1 | sort | uniq
Q: Website doesn't load. How will you investigate?
Check DNS resolution, server status, firewall, and web server logs. Use curl -v or ping to debug.
Q: Using sed command, how to remove first and last line of the file
sed '1d;$d' filename
Q: Explain DNS in simple words.
DNS translates domain names into IP addresses so browsers can load websites.
Q: Explain the complete flow of the request from client to server (OSI Model)
Request goes through layers: App → Transport (TCP) → Network (IP) → Data Link → Physical → Server, then reverse path for response.
Q: Explain the difference between Forward Proxy and Reverse Proxy
Forward Proxy hides client identity; used by clients.
Reverse Proxy hides backend servers; used by servers (like Nginx).
Q: User reports slowness in the app. How would you approach this?
Check server CPU, memory, DB, and network latency.
Use tools like top, ping, and logs to find root cause.
Q: Curl works with IP, fails with domain. Why?
DNS issue: domain name isn't resolving. Check /etc/resolv.conf or DNS records.
Q: Website returns 502 HTTP Status code. What can be the issue?
502 means Bad Gateway. Backend server might be down or misconfigured. Check Nginx/Apache and app logs.
Q: What is the difference between 0.0.0.0 and 127.0.0.1?
127.0.0.1 is localhost. 0.0.0.0 means "all interfaces". It's used to bind a service on all IPs.
Q: What is the difference between Public and Private Subnets?
Public subnet has internet gateway access. Private subnet doesn’t—used for backend servers.
Q: You accidentally created a private subnet instead of public. How will you fix it?
Attach an internet gateway, update route table with IGW, and assign public IP to the instance.
Q: What is Docker?
Docker is a containerization platform used to package applications and their dependencies into containers.
Q: Docker Container Exits Immediately, how will you troubleshoot?
Use docker logs <container-id> to check the error.
Ensure the container’s default command is long-running or attach an interactive shell.
Q: What is the purpose of EXPOSE in Dockerfile?
EXPOSE is documentation for which ports the container listens on.
It doesn’t publish the port — you still need -p or --publish when running the container.
Q: Port is Not Accessible on localhost even after Port mapping in Docker
Ensure port is published using -p hostPort:containerPort.
Check firewall and confirm app is bound to 0.0.0.0 and not localhost.
Q: Data lost when container stops and restarts, How will you fix it?
Use volumes for persistent storage:
-v /host/path:/container/path
Or named volumes using docker volume create.
Q: You made a code change, rebuilt the image, but the change isn't reflected?
Docker may be using cached layers.
Add --no-cache while building or verify that your COPY command runs after code changes.
Q: App Crashes with "Permission Denied" in Container but works fine on localhost
File permissions or user mismatch.
Use USER in Dockerfile or set proper permissions with chmod / chown.
Q: Docker host is running out of disk space. How do you clean up?
Use docker system prune to remove unused data.
Or more aggressively: docker system prune -a (removes all stopped containers, unused images, networks).
Q: How will you Debug a Live Container?
Use docker exec -it <container> /bin/bash or sh to inspect the container.
Check logs with docker logs <container>.
Q: Which container registry do you use in your organization?
We use Docker Hub / GitHub Container Registry / Amazon ECR / GitLab Registry depending on project needs.
Q: Explain the difference between CMD and Entrypoint in Docker
CMD provides default arguments to ENTRYPOINT or runs directly if no ENTRYPOINT.
ENTRYPOINT is not overridden when arguments are passed during docker run.
Q: What docker commands you use on day to day basis?
docker build, docker run, docker ps, docker exec, docker logs, docker stop, docker rm, docker images, docker system prune
Q: When will you forcefully remove a container and how?
When container is stuck or unresponsive.
Use docker rm -f <container-id> to force remove it.
Q: What is Kubernetes?
Kubernetes is an open-source platform for automating deployment, scaling, and operations of application containers.
Q: Explain Kubernetes Cluster Architecture
Kubernetes has a master-worker architecture:
- Control Plane: API Server, Controller Manager, Scheduler, etcd
- Nodes: run kubelet, kube-proxy, and containers via container runtime
Q: How various components of Kubernetes interact when you run kubectl apply (Pod)
1. `kubectl` sends request to API Server
2. API Server stores Pod spec in etcd
3. Scheduler assigns node
4. Kubelet creates pod on the node
5. Kube-proxy handles networking
Q: What is the purpose of services in Kubernetes?
Services provide stable networking to Pods, allowing internal/external access using DNS names, not pod IPs.
Q: Why is hardcoding Pod IP communication a bad practice?
Pod IPs are ephemeral. Hardcoding them breaks communication when pods restart or reschedule. Use Services or DNS.
Q: What are the types of services in Kubernetes?
1. ClusterIP (default, internal)
2. NodePort (external via node IP)
3. LoadBalancer (via cloud LB)
4. ExternalName (maps to external DNS)
Q: What are labels and selectors in Kubernetes?
Labels are key-value pairs to organize resources.
Selectors match labels to filter resources — used in services, deployments, etc.
Q: NodePort vs LoadBalancer service — which one and why?
Use LoadBalancer in cloud for automatic external IPs.
Use NodePort for local testing or basic setups.
Q: How are Kubernetes Services related to Kube Proxy?
kube-proxy maintains rules on each node to route traffic to backend pods using iptables or IPVS.
Q: Disadvantage of LoadBalancer service type?
Requires cloud provider, consumes IPs, and incurs cost for each service.
Q: What is a Headless Service and when to use it?
Headless service (clusterIP: None) skips load-balancing, returns individual pod IPs — used in StatefulSets or custom discovery.
Q: Can a Pod access a Service in another namespace?
Yes, use the full DNS: service-name.namespace.svc.cluster.local
Q: Restrict DB Pod access to only one app in the namespace?
Use NetworkPolicies to allow only selected Pod labels to access DB pod.
Q: Deployment strategy followed in your organization?
We use RollingUpdate strategy with maxUnavailable and maxSurge settings for zero downtime.
Q: Rollback strategy in your organization?
Use kubectl rollout undo deployment/<name>
Keep previous versions using Deployment history.
Q: Design a solution to avoid rollbacks?
Use canary deployments, liveness probes, and automated testing before full rollout to catch issues early.
Q: Deployment strategies used in past?
Blue-Green, Canary, and Rolling Updates depending on app criticality and traffic pattern.
Q: What is CoreDNS in Kubernetes?
CoreDNS is the cluster DNS server that resolves service and pod DNS names in Kubernetes.
Q: Node tainted as NoSchedule — can you still schedule a Pod?
Only if the Pod tolerates the taint using a toleration in its spec.
Q: Pod is in CrashLoopBackOff — how to fix it?
Check logs with kubectl logs, describe the pod, fix app errors or init issues, check health probes.
Q: Difference between liveness and readiness probes?
- Liveness: checks if app is running, restarts if fails
- Readiness: checks if app is ready to serve traffic
Q: Difference between Ingress and LoadBalancer?
LoadBalancer creates external IP per service.
Ingress routes multiple services through one IP based on path or host rules.
Q: App works with ClusterIP but fails with Ingress. Troubleshoot?
Check Ingress controller is installed.
Validate rules, service targets, and DNS setup.
Q: Why set up Ingress controller after creating Ingress?
Ingress resource is just a config. Ingress controller is needed to enforce it and route traffic.
Q: Use Ingress with in-house Load Balancer?
Yes, configure Ingress controller to use your load balancer as external traffic entry point.
Q: Deployment has 3 replicas but only 1 Pod is running — why?
Check resource limits, node taints, scheduling issues, and kubectl describe deployment for events.
Q: Pod mounts ConfigMap but changes aren’t reflected?
Reload app manually or restart pod — changes to ConfigMap don’t auto-refresh unless the app watches the file.
Q: How Node Affinity works and when to use?
Schedules pods on nodes matching specific labels. Used for hardware needs or zone-based placement.
Q: Difference between Node Affinity and Node Selector?
Node Selector is simple equality match.
Node Affinity supports advanced logic (e.g., soft/hard rules, multiple expressions).
Q: What is container runtime in Kubernetes?
It's the software that runs containers, e.g., containerd, CRI-O. Kubernetes uses it via the CRI interface.
Q: What is Jenkins?
Jenkins is an automation server used to build, test, and deploy code continuously with CI/CD.
Q: What is Terraform?
Terraform is an open-source IaC (Infrastructure as Code) tool that allows you to define and provision infrastructure using a high-level configuration language.
Q: What is Terraform and why is it used?
Terraform is an open-source IaC tool used to provision and manage infrastructure across cloud providers using declarative configuration files.
Q: What is the difference between Terraform and CloudFormation?
Terraform is cloud-agnostic, uses HCL, has a larger community, and supports modules. CloudFormation is AWS-specific, uses JSON/YAML.
Q: Explain Terraform workflow (init, plan, apply).
- terraform init: initializes config
- terraform plan: shows what will be changed
- terraform apply: applies the changes
Q: What is a Terraform state file?
It's a JSON file that maps your configuration to real infrastructure. It tracks resource IDs and metadata.
Q: How do you manage Terraform state in a team?
Use remote backends like S3 with state locking via DynamoDB to avoid conflicts and enable collaboration.
Q: What are Terraform providers?
Providers are plugins that allow Terraform to interact with APIs of cloud providers or services (e.g., AWS, Azure, GitHub).
Q: Difference between Terraform module and resource?
A resource is a single infrastructure object. A module is a reusable container for multiple resources grouped logically.
Q: What is Terraform taint and when would you use it?
terraform taint marks a resource to be destroyed and recreated on next apply. Useful when something gets corrupted.
Q: Lecture 116: How do you import existing infrastructure into Terraform?
Use terraform import <resource.type.name> <resource_id>. Then manually add the resource config to your .tf file.
Q: What is the purpose of output variables?
Output variables expose values (like IPs, URLs) after a run. They can be used by modules or external scripts.
Q: What is a backend in Terraform?
A backend determines how state is loaded and where it is stored. Examples: local, S3, Consul, etc.
Q: What are locals in Terraform and when to use them?
Locals define reusable expressions or values within a module. Use them for simplifying logic and avoiding repetition.
Q: How do you handle secrets in Terraform?
Avoid hardcoding secrets. Use environment variables, encrypted backends, or external secret managers like AWS Secrets Manager or Vault.
Q: How do you manage multiple environments in Terraform?
Use workspaces, separate state files, or directory structures with different variable values per environment (dev/staging/prod).
Q: What is the difference between count and for_each?
count is index-based and works with integers.
for_each is map/set-based and better for unique resource configurations.
Q: What is a data source in Terraform?
Data sources allow Terraform to fetch data from external systems (e.g., existing AWS resources) without managing them.
Q: How do you upgrade Terraform version safely in a project?
1. Update required_version in config
2. Run terraform init -upgrade
3. Check plan for changes and test in isolated environment
Q: Can two Terraform resources depend on each other?
Yes, but it may create a circular dependency. Use depends_on carefully to manage order without introducing loops.
